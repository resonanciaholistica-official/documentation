# 5.2 Guía de Conexión — Para servicios sin servidor propio (Cloudflare)

> **¿Quién usa esto?**  
> Si tu web está en Wix, Shopify, Webflow, Squarespace, o cualquier plataforma donde **no puedes subir archivos PHP**, esta guía es para ti.

---

## 📋 ¿Qué necesito?

| Requisito | ¿Lo tengo? |
|-----------|-------------|
| **Cuenta Cloudflare** | Es gratis — la creamos juntos aquí: https://dash.cloudflare.com/sign-up |
| **Tu dominio** | Debe estar gestionado por Cloudflare (te ayudamos a ponerlo) |

**Tiempo total: 20-30 minutos**

---

## 🧠 ¿Qué vamos a hacer?

Vamos a crear un "mini programa" (Cloudflare lo llama **Worker**) en los servidores de Cloudflare.  
Este Worker actuará como el puente entre tu web y la API de DBIO.

**Piensa en él como un portero**: tu web le habla a él, y él se encarga de hablar con nosotros de forma segura.

---

## 🚀 Paso a paso

### Paso 1: Crear una cuenta en Cloudflare (si no tienes)

1. Ve a https://dash.cloudflare.com/sign-up
2. Regístrate con tu correo electrónico
3. Confirma tu correo (te llegará un enlace de verificación)

> ✅ Listo. Ya tienes tu cuenta Cloudflare.

---

### Paso 2: Añadir tu dominio a Cloudflare

Si tu dominio NO está aún en Cloudflare, haz esto:

1. En el panel de Cloudflare, haz clic en **"Add a Site"**
2. Escribe tu dominio (ej: `tudominio.com`)
3. Selecciona el plan **Gratuito** (Free)
4. Cloudflare te dará **dos servidores DNS** (ej: `lara.ns.cloudflare.com` y `chad.ns.cloudflare.com`)

**Ahora ve a donde compraste tu dominio** (GoDaddy, Namecheap, etc.) y cambia los servidores DNS por esos dos.

> ⏱️ El cambio puede tardar unas horas. Normalmente son minutos.

---

### Paso 3: Crear el Worker

1. En el panel de Cloudflare, haz clic en **"Workers & Pages"**
2. Haz clic en **"Create application"**
3. Elige **"Start with Hello World"**
4. Ponle un nombre (ej: `dbio-bridge-tuempresa`)
5. **Reemplaza el código** que aparece por el que te daremos nosotros
6. Haz clic en **"Save and Deploy"**

> ✅ Worker creado y desplegado.

> 💡 **¿Tienes varios subdominios?** Tranquilo, el Worker acepta `admin.tudominio.com`, `app.tudominio.com`, `www.tudominio.com` y `tudominio.com` sin problema.

---

### Paso 4: Pegar el código

> 📎 **Nosotros te entregaremos el archivo del Worker ya configurado.** Solo tendrás que copiar su contenido.

1. Abre el archivo que te enviamos (`dbio-bridge-worker.js`) con el Bloc de notas o similar
2. **Selecciona todo** (Ctrl+E) y **cópialo** (Ctrl+C)
3. En el panel del Worker que dejaste abierto, **selecciona el código que hay** y **pégalo encima** (Ctrl+V)
4. Haz clic en **"Save and Deploy"**

> ✅ **IMPORTANTE**: No modifiques nada del código. Ya viene listo para usar.

---

### Paso 5: Obtener la URL de tu Worker

Al lado del nombre del Worker verás algo como:

```
https://dbio-bridge-tuempresa.nombre-de-tu-cuenta.workers.dev
```

**Copia esa dirección.** La vas a necesitar.

---

### Paso 6: Ajustar tu web para que use el Worker

Tu web necesita apuntar al Worker, no directamente a nuestra API.

Donde antes tu web decía algo como:

```javascript
apiUrl: 'https://api.institutopti.com/v1/agent'
```

Ahora debe decir:

```javascript
apiUrl: 'https://dbio-bridge-tuempresa.nombre-de-tu-cuenta.workers.dev/v1/agent'
```

> 💡 ¿Qué es `/v1/agent`? Es solo un ejemplo. Nosotros te diremos exactamente el final de la ruta que necesitas.  
> Consulte la documentación de la API para conocer los endpoints disponibles.

---

### Paso 7: Probar que funciona

Abre la consola de tu navegador (pulsa F12 → pestaña "Console") estando en tu web y pega esto:

```javascript
fetch('https://dbio-bridge-tuempresa.nombre-de-tu-cuenta.workers.dev/v1/test', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ petition: 'test' })
})
.then(r => r.json())
.then(console.log)
```

**Si todo está bien**, verás un mensaje como este:

```json
[
  {
    "status": "success",
    "message": "La conexión entre servidores es un éxito."
  }
]
```

> 🎉 ¡Conexión funcionando!

---

### Paso 8: (Opcional) Usar tu propio dominio

Si quieres una URL más profesional (ej: `https://api.tudominio.com`):

1. En el panel del Worker, ve a **"Triggers"** → **"Custom Domains"**
2. Haz clic en **"Add Custom Domain"**
3. Escribe el subdominio que quieras (ej: `api.tudominio.com`)
4. Listo

Ahora tu web apuntará a `https://api.tudominio.com/v1/...`

---

### Paso 9: ¡Listo!

```
Tus usuarios  →  Tu web  →  Worker de Cloudflare  →  API DBIO  →  Agente Inteligente
```

Nosotros gestionamos el resto.

---

## ❓ Algo no funciona?

### Error 403

```
Causa: Alguien está usando tu Worker desde otra web que no es la tuya.
       
Solución: No es un error tuyo. El Worker lo bloqueó automáticamente.
          Es una medida de seguridad.
```

### Error "No se pudo conectar"

```
Causa: El Worker no se desplegó bien.
       
Solución: Vuelve a "Workers & Pages" → haz clic en tu Worker → "Deploy"
```

### Error 1101

```
Causa: El código pegado tiene algún fallo.
       
Solución: Vuelve a copiar el código que te enviamos y pégalo de nuevo.
          Asegúrate de copiarlo todo.
```

### La URL workers.dev no carga

```
Solución: Espera unos minutos. Prueba en modo incógnito.
```

## ❓ Preguntas frecuentes

**¿Es gratis?**  
Sí, el plan gratuito de Cloudflare incluye 100.000 peticiones al día.

**¿Puedo tener un Worker para cada dominio?**  
Sí, cada dominio necesita el suyo.

**¿Se actualiza solo?**  
No. Si hay cambios, te avisaremos para que actualices.

**¿Consume recursos de mi web?**  
No. Todo ocurre en los servidores de Cloudflare.

**¿Es seguro?**  
Sí. La clave de acceso está en Cloudflare, no en el navegador del usuario.
