# 5.1 Guía de Conexión — Para servidores propios (PHP)

> **¿Quién usa esto?**  
> Si tienes tu propio servidor web (WordPress, cPanel, DirectAdmin, o cualquier hosting donde puedas subir archivos), esta guía es para ti.

---

## 📋 ¿Qué necesito?

| Requisito | ¿Lo tengo? |
|-----------|-------------|
| **PHP 7.4 o superior** | Tu hosting lo tiene (casi todos lo tienen) |
| **Conexión segura (HTTPS)** | Tu web debe cargar con https:// |
| **Acceso para subir archivos** | FTP, cPanel, o el administrador de archivos de tu hosting |

**Tiempo total: 5-10 minutos**

---

## 🚀 Paso a paso

### Paso 1: Recibir el archivo

Nosotros te enviaremos un archivo llamado **`dbio-bridge.php`** por correo electrónico.  
Este archivo ya viene preparado especialmente para ti, con tus datos de conexión incluidos.

> **Importante**: No modifiques el archivo. Ya está listo para usar.

---

### Paso 2: Subir el archivo a tu web

Conéctate a tu hosting (con FileZilla, el administrador de archivos de cPanel, o como lo hagas normalmente) y **sube el archivo a la carpeta raíz de tu web**.

La carpeta raíz se llama normalmente `public_html`, `www` o `htdocs`.  
Si no sabes cuál es, pregúntanos y te ayudamos.

Al final, el archivo tiene que quedar accesible en:

```
https://tudominio.com/dbio-bridge.php
```

> ✅ **Pruébalo**: Abre esa dirección en tu navegador. Si ves una página en blanco o un texto mínimo, está bien.

---

### Paso 3: Ajustar tu web para que use el bridge

Tu web necesita apuntar al bridge en lugar de ir directamente a nuestra API.

El cambio es muy sencillo. Donde antes tu web decía algo como:

```javascript
apiUrl: 'https://api.institutopti.com/v1/agent'
```

Ahora debe decir:

```javascript
apiUrl: 'https://tudominio.com/dbio-bridge.php/v1/agent'
```

> 💡 **¿Qué es `/v1/agent`?** Es un ejemplo. El final de la ruta dependerá de lo que quieras hacer (consultar el agente, enviar un mensaje, etc.). Nosotros te diremos exactamente qué ruta usar.

Si tu web tiene **varios subdominios** (ej: `admin.tudominio.com`, `app.tudominio.com`), no pasa nada — todos funcionan con el mismo archivo.

---

### Paso 4: Probar que funciona

Abre la consola de tu navegador (pulsa F12 → pestaña "Console") y pega esto:

```javascript
fetch('https://tudominio.com/dbio-bridge.php/v1/test', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ petition: 'test' })
})
.then(r => r.json())
.then(console.log)
```

**Si todo está bien**, verás un mensaje como este:

```json
[
  {
    "status": "success",
    "message": "La conexión entre servidores es un éxito."
  }
]
```

> 🎉 ¡Eso significa que la conexión funciona perfectamente!

---

### Paso 5: ¡Listo!

Ya está todo conectado. El flujo ahora es:

```
Tus usuarios  →  Tu web  →  dbio-bridge.php  →  API DBIO  →  Agente Inteligente
```

Nosotros nos encargamos del resto.

## ❓ Algo no funciona?

### Error 403 (Prohibido)

```
Causa: Alguien está usando tu bridge desde otra web que no es la tuya.
       
Solución: No pasa nada, el bridge lo bloqueó automáticamente. 
          Es una medida de seguridad.
```

### Error 500 o página en blanco al abrir dbio-bridge.php

```
Causa: Puede que tu hosting tenga una versión de PHP muy antigua.
       
Solución: Avísanos y lo revisamos.
```

### Error 404 (No encontrado)

```
Causa: El archivo no está en la carpeta correcta o tiene otro nombre.
       
Solución: Verifica que el archivo se llama exactamente "dbio-bridge.php"
          y que está en la raíz de tu web.
```

## ❓ Preguntas frecuentes

**¿Puedo cambiarlo de carpeta?**  
Sí, pero tendrías que ajustar la ruta en tu web.

**¿Consume muchos recursos de mi servidor?**  
No. Es un archivo muy pequeño que solo reenvía datos. No afecta al rendimiento.

**¿Qué hago si me sale un error?**  
Escríbenos y dinos exactamente qué mensaje de error viste. Te ayudamos al momento.
